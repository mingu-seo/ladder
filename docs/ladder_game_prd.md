# 📄 PRD

## 교육용 사다리 게임 웹서비스

---

## 1. 서비스 개요

### 1.1 서비스 목적

본 서비스는 강의 환경에서 강사가 다수의 학생을 대상으로  
자리 배치, 발표 순서, 실습 순서, 팀 배정 등을 **공정하고 직관적으로 결정**하기 위한  
웹 기반 사다리 게임 서비스이다.

### 1.2 핵심 가치

- 공정성: 결과가 임의로 조작될 수 없음을 시각적으로 전달
- 효율성: 수업 흐름을 끊지 않고 빠르게 결과 도출
- 직관성: 별도 설명 없이 즉시 사용 가능
- 몰입감: 애니메이션과 캐릭터를 통한 참여도 향상

---

## 2. 사용자 정의

### 2.1 주요 사용자

- **강사 1명**
  - 서비스 조작 주체
  - 대형 화면(PC/프로젝터) 기준 사용

### 2.2 결과 소비자

- **학생 다수**
  - 결과를 시각적으로 확인
  - 직접 조작은 하지 않음

---

## 3. 서비스 범위 (In / Out Scope)

### 3.1 포함 범위 (In Scope)

- 참가자 최대 **30명**
- 사다리 애니메이션
- 사다리 구조 랜덤 생성
- 결과 이미지(PNG) 저장
- 사용자별 캐릭터 자동 할당
- 완전 프론트엔드 기반 동작

### 3.2 제외 범위 (Out Scope)

- 로그인 / 회원가입
- 서버 / DB
- 결과 공유 URL
- 광고
- 팀 전용 모드 (결과 항목으로 대체)

---

## 4. 사용자 시나리오 (Primary Flow)

1. 강사가 서비스 페이지에 접속한다.
2. 참가자 수를 입력한다. (최대 30명)
3. 참가자 수만큼 이름 입력란이 자동 생성된다.
4. 결과 항목에 각 결과값을 입력한다.
5. 아래 중 하나를 선택한다.
   - 바로 실행
   - 랜덤 버튼 클릭 후 실행
6. 각 참가자가 상단에서 시작해 사다리를 따라 내려오는 애니메이션이 실행된다.
7. 각 참가자별로 매칭된 결과값이 표시된다.
8. 최종 결과 화면을 PNG 이미지로 저장한다.

---

## 5. 기능 요구사항

### 5.1 필수 기능 (MVP)

#### 5.1.1 참가자 설정

- 참가자 수 입력 (1 ~ 30명)
- 참가자 이름 입력
- 빈 값일 경우 자동 이름 생성
  - 예: 학생1, 학생2, …

#### 5.1.2 결과 항목 설정

- 참가자 수와 동일한 개수의 결과 항목 입력
- 결과 항목은 자유 텍스트

#### 5.1.3 사다리 생성

- 참가자 수에 맞춰 자동 생성
- 세로선은 고정
- 가로선은 랜덤 배치 가능

#### 5.1.4 실행

- 실행 버튼 클릭 시 애니메이션 시작
- 실행 이후 입력값 수정 불가

#### 5.1.5 애니메이션

- 참가자는 상단에서 시작
- 사다리를 따라 아래로 이동
- 이동 경로가 명확히 보이도록 표현
- 모든 참가자의 이동이 완료되면 결과 고정

#### 5.1.6 결과 표시

- 참가자 이름
- 자동 할당된 캐릭터
- 매칭된 결과 항목

---

### 5.2 선택 기능 (확장 기능)

#### 5.2.1 랜덤 버튼

- 참가자 순서 ❌ 변경 안 함
- 결과 순서 ❌ 변경 안 함
- **사다리의 가로선 구조만 랜덤으로 재생성**

#### 5.2.2 결과 이미지 저장

- 애니메이션 종료 후 최종 상태 기준
- 사다리 + 참가자 + 결과 포함
- PNG 파일로 저장

---

## 6. UX / UI 요구사항

### 6.1 UX 원칙

- 입력 → 실행 → 결과 흐름은 단방향
- 실행 이후 되돌릴 수 없음
- 결과가 “과정”을 통해 결정됨을 시각적으로 표현

### 6.2 UI 기준

- 단일 페이지 구성
- 대형 화면(프로젝터) 가독성 우선
- 참가자 수가 많을 경우 자동 스케일 또는 스크롤 처리

---

## 7. 상태 및 제약 조건

### 7.1 상태 제어

- 실행 이후:
  - 새로고침 시 경고 또는 차단
  - 뒤로가기 차단 또는 확인 팝업

### 7.2 랜덤 정책

- 결과는 재현 불필요
- 매 실행 시 완전 랜덤
- 동일 입력이라도 결과는 달라질 수 있음

---

## 8. 캐릭터 시스템

### 8.1 캐릭터 할당

- 참가자 입력 순서 기준 자동 할당
- 미리 정의된 캐릭터 세트 사용
- 사용자 수정 불가

### 8.2 목적

- 참가자 식별 보조
- 수업 몰입도 향상
- 결과 시각적 구분 강화

---

## 9. 기술적 가정 (기획 기준)

- 완전 프론트엔드 구현
- Canvas 또는 SVG 기반 사다리 렌더링
- 브라우저 로컬 상태만 사용
- 외부 API / 서버 호출 없음

## 12. 화면 구성 및 컴포넌트 설계 (AI Studio 전달용)

본 섹션은 UI 구현을 위한 화면 단위 설계 문서이며,  
기능 요구사항을 실제 화면과 컴포넌트 기준으로 분해한 내용이다.  
AI Studio 및 프론트엔드 구현 시 본 섹션을 기준으로 UI를 구성한다.

---

### 12.1 전체 화면 구조

본 서비스는 단일 페이지(Single Page) 구조로 구성된다.

화면은 아래 4개 영역으로 구분된다.

1. Header 영역
2. Settings Section (입력 영역)
3. Ladder Stage (사다리 애니메이션 영역)
4. Result Action Area (결과 저장 영역)

---

### 12.2 Header 영역

#### 목적

- 서비스 성격 즉시 인지
- 수업 중 시선 분산 최소화

#### 구성 요소

- 서비스 타이틀 텍스트

#### UI 규칙

- 단순한 텍스트 중심
- 고정 높이
- 스크롤 여부와 관계없이 항상 상단에 유지 가능

---

### 12.3 Settings Section (입력 영역)

본 영역은 **실행 전까지만 활성화**되며,  
실행 이후에는 모든 입력 요소가 비활성화된다.

---

#### 12.3.1 참가자 수 설정 컴포넌트

##### 구성

- 숫자 입력 필드
- 적용 버튼

##### 동작 규칙

- 입력 범위: 1 ~ 30
- 참가자 수 변경 시:
  - 참가자 이름 입력란 자동 생성/제거
  - 결과 항목 입력란 자동 동기화

---

#### 12.3.2 참가자 이름 입력 리스트

##### 구성

- 참가자 수만큼 생성되는 텍스트 입력 필드 목록
- 최대 30개

##### UX 규칙

- 기본값 자동 입력
  - 예: 학생1, 학생2, …
- Enter 키 입력 시 다음 입력 필드로 포커스 이동
- 빈 값은 자동 이름으로 처리

---

#### 12.3.3 결과 항목 입력 리스트

##### 구성

- 참가자 수와 동일한 개수의 텍스트 입력 필드
- 자유 텍스트 입력

##### 동작 규칙

- 참가자 수 변경 시 항상 1:1 개수 유지
- 실행 전까지만 수정 가능

---

#### 12.3.4 컨트롤 버튼 영역

##### 버튼 구성

- 랜덤 버튼
- 실행 버튼

##### 랜덤 버튼 규칙

- 참가자 순서 변경 ❌
- 결과 항목 순서 변경 ❌
- 사다리의 **가로선 구조만 랜덤으로 재생성**

##### 실행 버튼 규칙

- 클릭 시 사다리 애니메이션 시작
- 실행 이후:
  - 모든 입력 요소 비활성화
  - 컨트롤 버튼 비활성 또는 숨김 처리

---

### 12.4 Ladder Stage (사다리 애니메이션 영역)

#### 역할

- 서비스의 핵심 시각 영역
- 학생들의 주 시선 집중 영역

---

#### 12.4.1 렌더링 방식

- Canvas 또는 SVG 기반
- 참가자 수에 따라 가로 폭 자동 계산
- 세로 높이는 고정 또는 비율 기반

---

#### 12.4.2 사다리 구성 요소

- 상단 영역
  - 참가자 이름
  - 자동 할당된 캐릭터
- 중단 영역
  - 세로선
  - 가로선 (랜덤 생성)
- 하단 영역
  - 결과 항목 텍스트

---

#### 12.4.3 애니메이션 규칙

- 모든 참가자는 동시에 상단에서 시작
- 각 참가자는 자신의 경로를 따라 아래로 이동
- 이동 경로가 명확히 보이도록 시각적 강조 처리
- 애니메이션 종료 후:
  - 결과 상태 고정
  - 추가 애니메이션 없음

---

### 12.5 Result Action Area (결과 저장 영역)

#### 구성 요소

- 결과 이미지 저장 버튼

#### 동작 규칙

- 애니메이션 종료 후 버튼 활성화
- 현재 화면 상태를 그대로 캡처
- PNG 파일로 저장

#### 저장 이미지 포함 요소

- 참가자 이름
- 캐릭터
- 사다리 구조
- 결과 항목

---

### 12.6 전역 상태 규칙

#### 실행 이후 상태 제어

- 입력 필드 수정 불가
- 새로고침 시 경고 또는 차단
- 브라우저 뒤로가기 차단 또는 확인 팝업 표시

---

### 12.7 AI Studio 전달용 요약 가이드

- 본 UI는 강의실 대형 화면 사용을 전제로 한다.
- 입력 → 실행 → 결과 흐름은 단방향이다.
- 실행 이후에는 어떤 조작도 허용되지 않는다.
- 랜덤 기능은 사다리 구조만 변경한다.
